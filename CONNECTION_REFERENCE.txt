## 📊 Database Schema

### MySQL Database: `editning`

Only ONE table (minimal MySQL usage):

```sql
CREATE TABLE admins (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Why Only One Table?**
- Security: Password hashing in MySQL
- Simplicity: JSON for flexible data
- Backup: Easy JSON file backups
- Scalability: Mix best of both worlds

## 🔄 Data Flow

### Booking Submission
```
User fills form in book.php
    ↓
PHP validates data
    ↓
db-functions.php → addBooking()
    ↓
Data saved to bookings.json
    ↓
Success message shown to user
```

### Contact Form Submission
```
User fills form in contact.php
    ↓
PHP validates data
    ↓
db-functions.php → addContactMessage()
    ↓
Data saved to contact_messages.json
    ↓
Success message shown to user
```

### Admin Login
```
User enters credentials in login.php
    ↓
PHP queries MySQL admins table
    ↓
password_verify() checks password hash
    ↓
Session created
    ↓
Admin redirected to dashboard.php
```

### View Bookings (Admin)
```
Admin navigates to bookings_total.php
    ↓
Session verification (must be logged in)
    ↓
db-functions.php → getBookings()
    ↓
Reads bookings.json
    ↓
Displays table with all bookings
```

## 🚨 If Something Goes Wrong

### A) Can't Login to Admin
1. Check: Is `admins` table created?
2. Check: Is default user inserted?
3. Solution: Run `hosting-setup.sql` again

### B) Booking/Contact Forms Not Working
1. Check: Do bookings.json and contact_messages.json exist?
2. Check: Are file permissions 644?
3. Check: Is db-functions.php in root directory?
4. Solution: Create empty JSON files and set permissions

### C) Database Connection Error
1. Check: Are database credentials correct?
2. Check: Is database host correct? (not localhost for hosting)
3. Check: Is database existing? (Run hosting-setup.sql)
4. Solution: Verify all 4 values match hosting panel
